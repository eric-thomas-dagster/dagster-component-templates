# precisely_dis_dq_check

## Purpose

Wraps Precisely Data Integrity Suite (DIS) Data Quality rules as
N Dagster asset checks — one `@asset_check` per rule, individually
alertable, retriable, and visible in the catalog UI.

Auth follows DIS's documented OAuth2 client-credentials flow against the
global token endpoint; rule execution + result fetch hits the DQ service
base path (override both `token_url` / `dq_base_url` for non-global
tenants).

## How it works (runtime contract)

1. `POST {token_url}` with `client_id` + `client_secret` → access token
2. `POST {dq_base_url}/datasets/{dataset_id}/rules/run` with `{ruleIds: [...]}` → `run_id`
3. Poll `GET {dq_base_url}/runs/{run_id}` until `status=COMPLETE` (or `FAILED`/`ERROR`)
4. `GET {dq_base_url}/runs/{run_id}/results` → per-rule pass/fail + score
5. Emit one `AssetCheckResult` per rule with the score / pass-count / fail-count as metadata
