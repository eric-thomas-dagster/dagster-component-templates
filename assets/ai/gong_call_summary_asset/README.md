# Gong Call Summary Asset

Reads a DataFrame of Gong calls (typically produced by `GongCallsIngestionComponent`) and uses an OpenAI-compatible LLM to summarize each call transcript into a short paragraph + a list of action items.

Row-wise pattern (one API call per row) — mirrors `langchain_chain_asset` and `anthropic_llm`. Uses `response_format={"type": "json_object"}` to guarantee parseable JSON output.

## Installation

```bash
pip install openai pandas
```

## Output columns

Adds two columns to the upstream DataFrame:

- `call_summary` (rename via `summary_column`) — a 2-4 sentence summary string
- `action_items` (rename via `action_items_column`) — a `list[str]` of concrete follow-ups

Empty transcripts (when `skip_empty: true`) yield `""` and `[]` without an API call.

## Configuration

```yaml
type: dagster_component_templates.GongCallSummaryAssetComponent
attributes:
  asset_name: gong_call_summaries
  upstream_asset_key: gong_calls
  llm_provider: openai
  model: gpt-4o-mini
  api_key_env_var: OPENAI_API_KEY
  transcript_column: transcript
```

## Authentication

Set the env var identified by `api_key_env_var`:

```bash
export OPENAI_API_KEY="sk-..."
```

## Using a non-OpenAI backend

The OpenAI Python SDK talks to any OpenAI-compatible endpoint (Azure OpenAI, LiteLLM proxy, vLLM, Ollama, etc.). Point `base_url` at the compatible endpoint and set `api_key_env_var` accordingly:

```yaml
attributes:
  llm_provider: openai
  model: llama3.1-70b
  base_url: http://litellm:4000/v1
  api_key_env_var: LITELLM_MASTER_KEY
```
