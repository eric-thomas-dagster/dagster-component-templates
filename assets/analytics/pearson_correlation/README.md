# PearsonCorrelationComponent

Compute the Pearson correlation matrix across selected numeric columns of an
upstream DataFrame and emit the result as its own asset. Drop-in for Alteryx's
**Pearson Correlation** tool.

Two output shapes via `output_shape`:

- `long` (default) — one row per `(field1, field2)` pair with the correlation
  value. Easy to filter, join, or sort downstream.
- `wide` — the full correlation matrix as a square DataFrame (column names ==
  index labels), useful for heatmaps and matrix display.

`method` accepts `pearson` (default), `spearman` (rank-based), or `kendall`
(Kendall tau), matching `pd.DataFrame.corr()`'s options.

## Use Cases

- Multicollinearity diagnostics before fitting a regression model
- Feature-selection inputs for downstream `feature_importance` / model components
- Correlation snapshots tracked over time as a partitioned asset

  asset_name: feature_correlations
  upstream_asset_key: training_features
  columns: ["age", "income", "tenure_months", "monthly_charges"]

  method: pearson
  group_name: feature_engineering

## Inputs and Outputs

| | Type | Description |
|-|------|-------------|
| Input | `pd.DataFrame` | Upstream DataFrame containing the numeric columns to correlate |
| Output | `pd.DataFrame` | Long-form (`field1`, `field2`, `correlation`) or wide-form matrix, per `output_shape` |

## Notes

- If `columns` is omitted, every numeric dtype column is used.
- `min_periods` is forwarded to `pd.DataFrame.corr()` — pairs with fewer
  observations than `min_periods` return `NaN`.
- Non-numeric columns in `columns` raise a `TypeError` upfront.
