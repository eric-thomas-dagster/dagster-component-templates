# PineconeAssetComponent

Load vector embeddings into a [Pinecone](https://www.pinecone.io) index as a Dagster asset. Designed for RAG (Retrieval-Augmented Generation) pipelines where embedding generation and vector storage are separate, observable pipeline steps.

## Use case

In a typical RAG pipeline, embeddings are generated by one asset (calling OpenAI, Cohere, a local model, etc.) and must then be loaded into a vector store before the retrieval application can use them. `PineconeAssetComponent` makes the vector loading step a first-class Dagster asset so you can:

- **Observe every upsert** — vector count, namespace, and batch progress are recorded as Dagster materialisation metadata.
- **Gate retrieval applications** on a fresh index via `deps`, ensuring the RAG system never queries stale embeddings.
- **Schedule index refreshes** using Dagster sensors or schedules without writing boilerplate upsert code.
- **Provision new indexes** (`create_index`) as part of an automated setup pipeline.

## Prerequisites

```bash
pip install "pinecone-client>=3.0.0" pandas>=1.3.0
```

You will also need a Pinecone account and API key: <https://app.pinecone.io>

## RAG pipeline pattern

The canonical two-asset pattern:

```
embeddings_generator  →  product_embeddings_index
     (upstream)               (this component)
```

```yaml
# Component 1: generate embeddings (your custom asset or another component)
# Produces a DataFrame with columns: product_id, embedding, name, category, price

# Component 2: load into Pinecone
type: dagster_component_templates.PineconeAssetComponent
attributes:
  asset_name: product_embeddings_index
  api_key_env_var: PINECONE_API_KEY
  index_name: product-embeddings
  operation: upsert
  upstream_asset_key: product_embeddings   # waits for embeddings_generator
  id_column: product_id
  vector_column: embedding
  metadata_columns: [name, category, price]
  batch_size: 200
  namespace: production
  group_name: vector_store
```

The `upstream_asset_key` is automatically added to the asset's `deps`, so Dagster will refuse to materialise the Pinecone asset until the embeddings asset is up to date.

## Operations

### upsert (default)

Batch-upsert vectors from a DataFrame into the index. The DataFrame must contain at minimum the columns referenced by `id_column` and `vector_column`.

```yaml
operation: upsert
id_column: product_id
vector_column: embedding
metadata_columns: [name, category, price]
batch_size: 200
```

Asset metadata after materialisation: `index`, `vectors_upserted`, `namespace`, `batch_size`.

### create_index

Provision a new Pinecone serverless index. Run this once as a setup asset before any upsert assets.

```yaml
operation: create_index
index_name: product-embeddings
dimension: 1536        # match your embedding model output size
metric: cosine
cloud: aws
region: us-east-1
```

Pinecone indexes for `text-embedding-ada-002` (OpenAI) are 1 536 dimensions. For `text-embedding-3-small` use 1 536; for `text-embedding-3-large` use 3 072. Check your model's documentation.

### delete

Delete vectors by ID from the index. The DataFrame must contain the `id_column`.

```yaml
operation: delete
upstream_asset_key: products_to_remove
id_column: product_id
```

### query

Run a nearest-neighbour query using the first row of the source DataFrame as the query vector. Top-10 results and the top match ID/score are recorded as asset metadata. Useful for smoke-testing index quality in a pipeline.

```yaml
operation: query
upstream_asset_key: query_vector_asset
vector_column: embedding
namespace: production
```

## Namespaces

Pinecone namespaces partition vectors within a single index — useful for multi-tenant applications or staging/production isolation:

```yaml
namespace: production    # or: staging, tenant_123, etc.
```

All operations (upsert, delete, query) respect the `namespace` field. Omit it to use the default namespace.

## Metadata filtering

Storing metadata alongside vectors enables filtered similarity search — for example, "find the 10 most similar products in the 'electronics' category":

```yaml
metadata_columns: [category, brand, price_tier]
```

These columns are included verbatim from the source DataFrame. Values are cast to `str` if they are not a native Pinecone metadata type (`str`, `int`, `float`, `bool`).

In your retrieval application, pass a metadata filter to `index.query`:

```python
index.query(
    vector=query_embedding,
    top_k=10,
    filter={"category": {"$eq": "electronics"}},
)
```

## Batch sizing for large datasets

The default `batch_size` of 100 is conservative.  For faster upserts:

- Pinecone recommends batches of 100–1 000 vectors.
- Each vector's byte size depends on its dimension: 1 536-dim float32 vectors are ~6 KB each.
- Pinecone's per-request limit is 2 MB. A batch of 200 1 536-dim vectors is ~1.2 MB — a safe ceiling.

```yaml
batch_size: 200   # safe for 1536-dim embeddings
```

Progress is logged after every batch so you can track upsert speed in the Dagster run log.

## Loading vectors from a database

As an alternative to an upstream Dagster asset, vectors can be read directly from a relational database:

```yaml
source_database_url_env_var: EMBEDDINGS_DB_URL
source_query: "SELECT product_id, embedding, name, category FROM product_embeddings WHERE updated_at > NOW() - INTERVAL '1 day'"
id_column: product_id
vector_column: embedding
metadata_columns: [name, category]
```

The database URL is read from the environment variable at runtime. Any SQLAlchemy-compatible database is supported.

## Asset dependencies

```yaml
attributes:
  asset_name: product_embeddings_index
  deps:
    - product_embeddings           # explicit upstream
    - embedding_model_checkpoint   # also wait for this
  ...
```

Downstream assets (e.g. a RAG evaluation harness, an API gateway smoke test) can declare:

```yaml
deps:
  - product_embeddings_index
```

## Reference

| Field | Type | Default | Description |
|---|---|---|---|
| `asset_name` | `str` | required | Dagster asset key |
| `api_key_env_var` | `str` | required | Env var with Pinecone API key |
| `index_name` | `str` | required | Pinecone index name |
| `operation` | `str` | `"upsert"` | `upsert`, `delete`, `query`, `create_index` |
| `upstream_asset_key` | `str` | `None` | Upstream asset providing a vectors DataFrame |
| `source_database_url_env_var` | `str` | `None` | Env var with SQLAlchemy DB URL |
| `source_table` | `str` | `None` | Table to read from (DB source) |
| `source_query` | `str` | `None` | SQL query to fetch vectors (DB source) |
| `id_column` | `str` | `"id"` | DataFrame column for vector IDs |
| `vector_column` | `str` | `"embedding"` | DataFrame column containing embedding arrays |
| `metadata_columns` | `list[str]` | `None` | Columns to store as Pinecone metadata |
| `namespace` | `str` | `None` | Pinecone namespace |
| `batch_size` | `int` | `100` | Vectors per upsert batch |
| `dimension` | `int` | `None` | Vector dimension (required for `create_index`) |
| `metric` | `str` | `"cosine"` | Distance metric for `create_index` |
| `cloud` | `str` | `"aws"` | Cloud provider for serverless index |
| `region` | `str` | `"us-east-1"` | Cloud region for serverless index |
| `group_name` | `str` | `"vector_store"` | Dagster asset group |
| `description` | `str` | `None` | Asset description |
| `deps` | `list[str]` | `None` | Upstream asset keys |
