# argo_workflow_trigger

## Purpose

Materializable Dagster asset that submits an [Argo Workflow](https://argo-workflows.readthedocs.io/) via the Argo Server REST API. Pairs with `argo_workflow_sensor` (in observation mode) for the submit‚Üíwatch loop.

Use this for **Case B** ‚Äî Dagster owns the schedule, fires the workflow, optionally watches for completion. For **Case A** (Argo owns the schedule), use `external_argo_workflow` + `argo_workflow_sensor` only. Don't mix on the same `asset_key`.

## Two submission modes ‚Äî set exactly one

### Submit from a WorkflowTemplate (most common)

```yaml
type: dagster_community_components.ArgoWorkflowTriggerComponent
attributes:
  asset_key: argo/etl/nightly_aggregation
  workflow_template: nightly-aggregation        # pre-registered on the cluster
  namespace: argo
  parameters:
    INPUT_DATE: "2026-05-27"
    REGION: us-east-1
  host_env_var: ARGO_SERVER_URL
  token_env_var: ARGO_TOKEN
```

POSTs to `/api/v1/workflows/{namespace}/submit` with the `WorkflowTemplate` kind. Argo generates the resulting Workflow name server-side (usually `nightly-aggregation-<5-char-suffix>`) ‚Äî the asset captures it as `argo_workflow_name` in materialization metadata.

### Submit an inline manifest

```yaml
attributes:
  asset_key: argo/etl/oneoff_backfill
  workflow_manifest:
    apiVersion: argoproj.io/v1alpha1
    kind: Workflow
    metadata:
      generateName: oneoff-backfill-
    spec:
      entrypoint: main
      templates:
        - name: main
          container:
            image: alpine
            command: ["echo", "hi"]
  namespace: argo
```

POSTs the full Workflow YAML to `/api/v1/workflows/{namespace}`. Useful for one-off workflows that aren't pre-registered as templates.

## Pairs with

`argo_workflow_sensor` in observation mode ‚Äî same `asset_key`, sensor watches the workflow name returned by the trigger and emits `AssetObservation` on terminal `Succeeded` / `Skipped`. The trigger's `AssetMaterialization` says "Dagster submitted the workflow"; the sensor's `AssetObservation` says "Argo confirmed terminal success."
