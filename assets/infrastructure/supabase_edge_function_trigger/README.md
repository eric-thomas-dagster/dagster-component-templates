# Supabase Edge Function Trigger

Materializable Dagster asset that invokes a [Supabase Edge Function](https://supabase.com/docs/guides/functions) via `POST https://<project-ref>.supabase.co/functions/v1/<function-name>`.

Uses the `SupabaseResource` client's `.functions.invoke(name, {"body": ...})` helper, which handles the `Authorization: Bearer <key>` + `apikey` headers automatically.

## Installation

```bash
pip install "supabase>=2.0.0" "requests>=2.28"
```

Requires a `SupabaseResource` (see `resources/supabase_resource`).

## Configuration

```yaml
type: dagster_component_templates.SupabaseEdgeFunctionTriggerComponent
attributes:
  asset_key: supabase/send_welcome_email
  function_name: send-welcome-email
  resource_name: supabase_resource
  payload:
    user_id: "abc-123"
    template: "welcome_v2"
  wait_for_result: true
```

## Notes

- The `service_role` key on the resource lets you invoke functions that require elevated privileges (e.g., functions gated by `verify_jwt = false` in the config still receive the `apikey` header).
- Response is parsed as JSON if possible; otherwise the raw text (up to 5000 chars) is stored in `response_text` metadata.
- Supabase Edge Functions are HTTP-synchronous; `wait_for_result: false` still makes the HTTP call — it just skips response capture.
