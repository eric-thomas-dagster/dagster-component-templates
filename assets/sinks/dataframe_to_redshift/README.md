# DataFrame to Redshift

Write a pandas DataFrame to an Amazon Redshift table using SQLAlchemy and psycopg2.

## Overview

This component receives a DataFrame from an upstream Dagster asset and writes it to a Redshift table. It uses the standard PostgreSQL-compatible SQLAlchemy dialect (`postgresql+psycopg2`) which is fully supported by Redshift.

## Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `asset_name` | `str` | required | Output Dagster asset name |
| `upstream_asset_key` | `str` | required | Upstream asset key providing a DataFrame |
| `table` | `str` | required | Destination Redshift table name |
| `schema` | `str` | `"public"` | Redshift schema |
| `host_env_var` | `str` | `"REDSHIFT_HOST"` | Env var containing Redshift host |
| `port` | `int` | `5439` | Redshift port |
| `database_env_var` | `str` | `"REDSHIFT_DATABASE"` | Env var containing Redshift database name |
| `user_env_var` | `str` | `"REDSHIFT_USER"` | Env var containing Redshift user |
| `password_env_var` | `str` | `"REDSHIFT_PASSWORD"` | Env var containing Redshift password |
| `if_exists` | `str` | `"replace"` | Behavior when the table exists: `replace`, `append`, or `fail` |
| `chunksize` | `int` | `10000` | Number of rows per write batch |
| `group_name` | `Optional[str]` | `None` | Dagster asset group name |

## Example YAML

```yaml
type: dagster_component_templates.DataframeToRedshiftComponent
attributes:
  asset_name: redshift_customers
  upstream_asset_key: processed_customers
  table: customers
  schema: analytics
  host_env_var: REDSHIFT_HOST
  port: 5439
  database_env_var: REDSHIFT_DATABASE
  user_env_var: REDSHIFT_USER
  password_env_var: REDSHIFT_PASSWORD
  if_exists: replace
  chunksize: 10000
  group_name: warehouse_sinks
```

## Authentication / Credentials

Set the following environment variables:

```bash
export REDSHIFT_HOST="my-cluster.abc123.us-east-1.redshift.amazonaws.com"
export REDSHIFT_DATABASE="dev"
export REDSHIFT_USER="awsuser"
export REDSHIFT_PASSWORD="supersecret"
```

You can use IAM-based authentication by creating a Redshift user with IAM credentials and setting the password to the temporary password generated by `redshift get-cluster-credentials`.

## if_exists Options

- `replace` — drops the existing table and recreates it from the DataFrame schema.
- `append` — inserts rows into the existing table without removing prior data.
- `fail` — raises an error if the table already exists.

## Requirements

dagster
