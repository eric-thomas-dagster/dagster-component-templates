# DataFrame to Redshift

Write a pandas DataFrame to an Amazon Redshift table using SQLAlchemy and psycopg2.

## Purpose

This component receives a DataFrame from an upstream Dagster asset and writes it to a Redshift table. It uses the standard PostgreSQL-compatible SQLAlchemy dialect (`postgresql+psycopg2`) which is fully supported by Redshift.

## Authentication / Credentials

Set the following environment variables:

```bash
export REDSHIFT_HOST="my-cluster.abc123.us-east-1.redshift.amazonaws.com"
export REDSHIFT_DATABASE="dev"
export REDSHIFT_USER="awsuser"
export REDSHIFT_PASSWORD="supersecret"
```

You can use IAM-based authentication by creating a Redshift user with IAM credentials and setting the password to the temporary password generated by `redshift get-cluster-credentials`.

## if_exists Options

- `replace` — drops the existing table and recreates it from the DataFrame schema.
- `append` — inserts rows into the existing table without removing prior data.
- `fail` — raises an error if the table already exists.
