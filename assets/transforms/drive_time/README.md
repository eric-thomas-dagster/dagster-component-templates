# DriveTimeComponent

Compute drive-time isochrones (polygons reachable within N minutes by car,
foot, or bike) for each row in an input DataFrame. Drop-in for Alteryx's
**Drive Time** and **Trade Area** (when configured for drive-time vs
straight-line buffer).

Provider-agnostic at the field level — pick one of:

| Provider | API / source | Auth |
|----------|-------------|------|
| `openrouteservice` (default) | openrouteservice.org Isochrones API | Free tier with API key (~40 req/min) |
| `google` | Google Maps Distance Matrix probe (8-bearing convex hull) | Billing-enabled API key |
| `mapbox` | Mapbox Isochrone API | API key |
| `osrm` | Self-hosted OSRM table-service probe | None (set `osrm_base_url`) |

The output is a `Polygon` per row in `geometry_column`, replacing the input
point (you typically pair this with `points_from_latlon` upstream to build
the points). Downstream `geo_overlay`, `spatial_join`, or `geo_buffer` then
operate on the isochrone polygons.

## Use Cases

- Trade-area analysis using realistic drive times (not crow-flies distance)
- "Customers within a 15-minute drive of this store"
- Site selection — score candidate locations by population reachable within
  a target drive time
- Service-coverage planning — find areas not within a fixed drive of any
  facility (`drive_time` per facility, `geo_overlay` union)

## Example

```yaml
type: dagster_community_components.DriveTimeComponent
attributes:
  asset_name: store_drive_15min
  upstream_asset_key: store_locations_geo
  drive_minutes: 15
  provider: openrouteservice
  api_key_env_var: OPENROUTESERVICE_API_KEY
  profile: driving-car
  geometry_column: geometry
  group_name: geospatial
```

OSRM (self-hosted) example:

```yaml
type: dagster_community_components.DriveTimeComponent
attributes:
  asset_name: store_drive_10min_osrm
  upstream_asset_key: store_locations_geo
  drive_minutes: 10
  provider: osrm
  osrm_base_url: http://osrm.internal:5000
  profile: driving-car
```

## Notes

- For inputs without geometry, set `latitude_column` + `longitude_column`
  and the component builds the Point internally — no separate
  `points_from_latlon` step required.
- `profile` accepts `driving-car` (default), `foot-walking`,
  `cycling-regular`. Provider-specific names are normalized.
- All providers respect rate limits internally. For large batches, expect
  the openrouteservice free tier to be the slowest (~40 req/min).
- `api_key_env_var` defaults to `OPENROUTESERVICE_API_KEY`; override for
  google (`GOOGLE_MAPS_API_KEY`) or mapbox (`MAPBOX_ACCESS_TOKEN`).
- `osrm` requires no auth — pass your self-hosted base URL via
  `osrm_base_url`.
