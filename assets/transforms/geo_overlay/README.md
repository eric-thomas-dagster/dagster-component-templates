# GeoOverlayComponent

Combine two geometry-bearing DataFrames using a set-theoretic spatial
operation. Drop-in for Alteryx's **Spatial Process** tool.

Supported `how` values (matching GeoPandas):

- `intersection` — only where both inputs overlap (the default).
- `union` — everywhere either input has coverage; attributes from both
  sides are preserved.
- `difference` — left minus right (areas of left not covered by right).
- `symmetric_difference` — xor; areas covered by exactly one input.
- `identity` — every part of left, split where right overlaps.

The output is one row per overlay-result feature, carrying attributes from
both inputs. On column-name collisions, suffixes `_1` / `_2` are appended
(GeoPandas default).

## Use Cases

- Intersect a service-area polygon with a customer-points polygon to find
  who falls inside each region
- Trade-area gap analysis (`difference` between competitor trade areas and
  your own)
- Combine zoning layers across districts (`union`)
- Risk modeling — overlay flood zones with building footprints

## Example

```yaml
type: dagster_community_components.GeoOverlayComponent
attributes:
  asset_name: customers_in_trade_areas
  left_asset_key: store_trade_areas
  right_asset_key: customer_locations_geo
  how: intersection
  keep_geom_type: true
  geometry_column: geometry
  group_name: geospatial
```

## Notes

- Both inputs must use the same CRS. Reproject upstream first if they
  don't (`gpd.GeoDataFrame.to_crs(...)`).
- `keep_geom_type: true` (default) prevents the GeoPandas quirk where
  polygon-polygon overlays sometimes leave stray points or lines.
- For point-in-polygon joins specifically, prefer `spatial_join` — it's
  much faster than `overlay(how='intersection')` on point inputs.
