# GeoSimplifyComponent

Reduce the vertex count of each geometry using Shapely's Douglas-Peucker
simplification. Drop-in for Alteryx's **Generalize** tool.

Higher `tolerance` removes more vertices (smoother shape, less fidelity);
lower `tolerance` preserves more detail. The right value depends on your
CRS units: for `EPSG:4326` (lat/lon degrees) `0.001` is roughly 100 m at
the equator; for `EPSG:3857` (Web Mercator meters) use the desired meter
threshold directly.

## Use Cases

- Lighten complex polygons for web-map plotting
- Shrink GeoJSON / tile sizes for static map distribution
- Speed up downstream spatial joins (`spatial_join`, `geo_overlay`) where
  exact boundaries aren't needed
- Pre-process coastline / boundary layers to a target detail level

## Example

```yaml
type: dagster_community_components.GeoSimplifyComponent
attributes:
  asset_name: simplified_districts
  upstream_asset_key: school_districts_geo
  tolerance: 0.001
  preserve_topology: true
  geometry_column: geometry
  group_name: geospatial
```

## Notes

- `preserve_topology: true` (default) keeps the geometry valid (no
  self-intersections) at a small performance cost. Set to `false` when
  you control the inputs and care about throughput.
- Point geometries are unchanged (nothing to simplify).
- The component logs the average vertex-count reduction per geometry.
