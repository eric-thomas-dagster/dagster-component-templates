# PointsFromLatLonComponent

Build Shapely `Point` geometries from a DataFrame's latitude / longitude
columns and emit a downstream-friendly geometry-bearing DataFrame. Drop-in
for Alteryx's **Create Points** tool.

Output is compatible with both `pd.DataFrame` consumers (the geometry column
is a Shapely object) and `gpd.GeoDataFrame` consumers (CRS is set on the
geometry column). Downstream `geo_buffer`, `geo_overlay`, `geo_simplify`,
`drive_time`, `spatial_join`, and `nearest_neighbors` components all accept
the output directly.

## Use Cases

- Convert a CSV / database table of lat/lon pairs into a geometry layer
- First step of any geospatial pipeline whose source is tabular GPS data
- Materialize raw GPS coordinates into a stable point layer used by many downstreams

## Example

```yaml
attributes:
  asset_name: customer_locations_geo
  upstream_asset_key: customer_locations
  longitude_column: lng
  latitude_column: lat
  geometry_column: geometry
  crs: EPSG:4326
  drop_invalid: true
  group_name: geospatial
```

## Inputs and Outputs

| | Type | Description |
|-|------|-------------|
| Input | `pd.DataFrame` | Must contain the `longitude_column` and `latitude_column` |
| Output | `pd.DataFrame` | All original columns plus `geometry_column` of Shapely `Point`s |

## Notes

- `crs` defaults to `EPSG:4326` (WGS84) — what GPS, mobile devices, and web
  maps emit. If your source uses something else (state plane, UTM, Web
  Mercator), set `crs` accordingly so downstream geo math is correct.
- With `drop_invalid: true` (default), rows where either column is `NaN` or
  non-numeric are silently dropped and a log line reports the count. With
  `false`, those rows survive with a `None` geometry — useful when you need
  to preserve row count for joins.
- Lon comes first in Shapely (`Point(x, y)` is `Point(lon, lat)`); this
  component handles that mapping for you.
