# precisely_address_verify

## Purpose

Verify + standardize an address column via the Precisely Address Verify
(a.k.a. Geo Addressing) REST API. Distinct from the generic `geocoder`
component: that one returns lat/lng from Nominatim/Google/HERE; this one
returns Precisely's CASS-certified standardized address plus match-quality
scoring ŌĆö what mailing-list deliverability, USPS Move Update, and address-
parsing use cases actually need.

The component reads an address column (or parsed address parts) from an
upstream DataFrame and writes verified columns next to the source. The
source column is never mutated.

## Output columns

Added to the right of the upstream DataFrame, prefixed by `output_prefix`
(default `av_`):

| Column | Meaning |
|---|---|
| `{prefix}status` | `AV` (verified) / `AC` (corrected) / `IC` (invalid) / `EC` (error) |
| `{prefix}standardized` | Single-line standardized address |
| `{prefix}match_score` | 0ŌĆō100 confidence |
| `{prefix}country` | ISO 3166-1 alpha-3 |
| `{prefix}postal_code` | Normalized postal/ZIP |
| `{prefix}latitude` / `{prefix}longitude` | If returned by tier |

Materialization metadata exposes `row_count`, `verified_count`, and
`verification_rate` (`AV + AC / total`) so you can dashboard
deliverability over time.

## Example YAML

```yaml
type: dagster_community_components.PreciselyAddressVerifyComponent
attributes:
  asset_name: verified_customer_addresses
  upstream_asset_key: customer_raw_addresses
  address_column: full_address
  default_country: USA
  output_prefix: av_
  client_id_env_var: PRECISELY_CLIENT_ID
  client_secret_env_var: PRECISELY_CLIENT_SECRET
  batch_size: 100
```

If your upstream DataFrame has the address parsed into multiple columns,
swap `address_column` for `address_line_columns`:

```yaml
attributes:
  asset_name: verified_customer_addresses
  upstream_asset_key: customer_raw_addresses
  address_line_columns:
    addressLines: street
    city: city
    stateProvince: state
    postalCode: zip
  default_country: USA
  output_prefix: av_
```
