# Record ID Component

Add a unique sequential identifier to each row in a DataFrame. Supports configurable start values, step increments, optional sorting before assignment, group-scoped numbering, and string prefixes.

This component uses Dagster's IO manager to pass DataFrames between assets. No IO manager configuration is required for local development — Dagster's default FilesystemIOManager handles serialization automatically.

## Fields

| Field | Type | Default | Description |
|---|---|---|---|
| `asset_name` | `str` | required | Output Dagster asset name |
| `upstream_asset_key` | `str` | required | Upstream asset key providing a DataFrame |
| `output_column` | `str` | `"record_id"` | Name of the ID column to add |
| `start` | `int` | `1` | Starting value for the ID sequence |
| `step` | `int` | `1` | Increment between consecutive IDs |
| `sort_by` | `Optional[str]` | `None` | Sort by this column before assigning IDs |
| `sort_ascending` | `bool` | `True` | Sort direction when `sort_by` is specified |
| `group_by` | `Optional[List[str]]` | `None` | Assign IDs within groups, restarting at `start` per group |
| `id_prefix` | `Optional[str]` | `None` | String prefix, e.g. `"REC-"` produces `"REC-1"`, `"REC-2"` |
| `group_name` | `Optional[str]` | `None` | Dagster asset group name |

## Example YAML

```yaml
type: dagster_component_templates.RecordIdComponent
attributes:
  asset_name: orders_with_id
  upstream_asset_key: processed_orders
  output_column: order_record_id
  start: 1000
  step: 1
  sort_by: order_date
  sort_ascending: true
  id_prefix: "ORD-"
  group_name: enrichment
```

## Sorting Before ID Assignment

When `sort_by` is specified, the DataFrame is sorted and the index is reset before IDs are assigned. This ensures that the ID sequence reflects a meaningful order (e.g. chronological, alphabetical).

## Group-Level IDs

When `group_by` is specified, IDs restart at `start` within each group. Row 0 of each group gets ID `start`, row 1 gets `start + step`, and so on. This is implemented using `groupby(...).cumcount()`.

## Prefixed IDs

When `id_prefix` is set, the numeric ID is converted to a string and the prefix is prepended. With `start=1000` and `id_prefix="ORD-"`, the first row receives `"ORD-1000"`.

## Notes

- If both `sort_by` and `group_by` are set, sorting occurs first, then group-level IDs are assigned within the sorted order.
- The numeric IDs generated by `group_by` mode are `0 * step + start` for the first row in each group, so `step` affects the increment but `start` always applies.
- The ID column is always added to the output DataFrame; existing columns are not modified.
