# Warehouse Formula (Alteryx "Formula In-DB" equivalent)

Add or replace columns via inline SQL expressions, executed in the warehouse via CTAS. No data through Python.

The warehouse-side counterpart to:
- pandas: [`formula`](../formula) — single-column expressions
- polars: [`polars_pipeline`](../polars_pipeline) op `with_columns`
- pyspark: [`pyspark_pipeline`](../pyspark_pipeline) op `with_columns`
- snowpark: [`snowpark_pipeline`](../snowpark_pipeline) op `with_columns`

## YAML

```yaml
type: dagster_component_templates.WarehouseFormulaComponent
attributes:
  asset_name: orders_enriched
  database_url: snowflake://...
  dialect: snowflake
  upstream_table: raw.orders
  output_table: analytics.orders_enriched
  expressions:
    net_amount:     "amount - tax"
    is_high_value:  "CASE WHEN amount > 1000 THEN TRUE ELSE FALSE END"
    order_year:     "EXTRACT(YEAR FROM created_at)"
    running_total:  "SUM(amount) OVER (PARTITION BY customer_id ORDER BY created_at)"
  keep_existing: true              # SELECT *, <expressions>
  mode: replace
  deps: [raw_orders_in_warehouse]
```

## Expression semantics

The expression body is **opaque SQL** — anything the dialect understands goes through verbatim:

- Arithmetic: `amount - tax`, `price * quantity`
- CASE / COALESCE / NULLIF: `CASE WHEN x > 0 THEN 'pos' ELSE 'neg' END`
- Date functions: `EXTRACT(YEAR FROM created_at)`, `DATEDIFF('day', a, b)`
- String functions: `UPPER(name)`, `CONCAT(first, ' ', last)`, `REGEXP_REPLACE(...)`
- Window functions: `SUM(amount) OVER (PARTITION BY customer_id ORDER BY ts)`
- Subqueries: `(SELECT MAX(rate) FROM fx WHERE date = orders.date) * amount`
- Dialect-specific: Snowflake `OBJECT_INSERT()`, BigQuery `ARRAY_AGG()`, etc.

## Output column selection

| Setting | Behavior |
|---|---|
| `keep_existing: true` (default) | `SELECT *, expr1 AS col1, expr2 AS col2 ...` — adds the new cols alongside originals |
| `keep_existing: false` + `keep_columns: [a, b]` | `SELECT a, b, expr1 AS col1, ...` — keeps only the named originals plus the new cols |
| `keep_existing: false`, no `keep_columns` | `SELECT expr1 AS col1, expr2 AS col2 ...` — ONLY the computed columns |

## Generated SQL

```sql
CREATE OR REPLACE TABLE analytics.orders_enriched AS
SELECT *,
  (amount - tax) AS "net_amount",
  (CASE WHEN amount > 1000 THEN TRUE ELSE FALSE END) AS "is_high_value",
  (EXTRACT(YEAR FROM created_at)) AS "order_year",
  (SUM(amount) OVER (PARTITION BY customer_id ORDER BY created_at)) AS "running_total"
FROM "raw"."orders"
```

## Siblings for special cases

- Same formula applied to many columns (e.g., `UPPER(TRIM(...))` on every string column) → [`warehouse_multi_field_formula`](../warehouse_multi_field_formula)
- Row-relative formulas (running totals, LAG/LEAD, ranks) as a declarative DSL → [`warehouse_multi_row_formula`](../warehouse_multi_row_formula). Window functions also work inline here via opaque SQL.
