# external_argo_workflow

## Purpose

Declare an [Argo Workflow](https://argo-workflows.readthedocs.io/) (Kubernetes-native workflow engine) as an external asset in the Dagster catalog. Declare-only `AssetSpec` — no execution function. Argo runs the workflow on its own schedule; Dagster observes via the paired `argo_workflow_sensor`.

Use this for **Case A** — Argo owns the schedule, Dagster catalogs + observes. For **Case B** (Dagster owns the schedule), use `argo_workflow_trigger` instead. Don't mix on the same `asset_key`.

## Example YAML

```yaml
type: dagster_community_components.ExternalArgoWorkflowAsset
attributes:
  asset_key: argo/etl/nightly_aggregation
  workflow_template: nightly-aggregation
  namespace: argo
  argo_ui_url: https://argo.mycompany.com
  group_name: argo
```

## Pairs with

- `argo_workflow_sensor` — same `asset_key`, sensor emits `AssetMaterialization` on terminal `Succeeded` / `Skipped`.
- (alternative) `argo_workflow_trigger` if Dagster owns the schedule — same asset_key shape, but the trigger emits the materialization itself.
