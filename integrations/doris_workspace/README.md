# doris_workspace

Auto-discover everything in an [Apache Doris](https://doris.apache.org/) database — tables (DUP / UNIQUE / AGG models), views, and async materialized views — and emit one observable Dagster asset per discovered entity. Analogous to `snowflake_workspace` but scoped to Doris's narrower surface (no streams / stages / alerts / openflow).

Each asset is an `@observable_source_asset` that polls `information_schema.tables` (and `information_schema.materialized_views` for async MVs) on every observation tick. The ObserveResult carries a change-sensitive `data_version` signature so downstream `AutomationCondition.eager()` only re-fires when row count or update time actually changes — not on every poll.

## Example — minimal

```yaml
type: dagster_component_templates.DorisWorkspaceComponent
attributes:
  host: doris.mycompany.com
  database: analytics
  username: dagster_reader
  password_env_var: DORIS_PASSWORD
```

Discovers every table / view / MV in every non-system schema. Asset keys are `<schema>__<name>` (sanitized to `[a-zA-Z0-9_]`).

## Example — filtered + grouped

```yaml
type: dagster_component_templates.DorisWorkspaceComponent
attributes:
  host: doris.mycompany.com
  database: analytics
  username: dagster_reader
  password_env_var: DORIS_PASSWORD
  schemas: [sales, marketing]
  include_patterns: ["fact_*", "dim_*"]
  exclude_patterns: ["*_staging", "*_tmp"]
  group_name: doris_analytics
  owners: ["data-platform@mycompany.com"]
```

## `data_version` semantics

Each `ObserveResult` carries `DataVersion(f"{row_count}:{last_update}")` (tables) or `DataVersion(f"{row_count}:{last_refresh}")` (MVs). When you bind a downstream asset with `AutomationCondition.eager()`, it only re-fires when the source entity actually changes — not on every observation poll. Use a schedule (e.g. every 5 min) to drive observation frequency.

## Differences vs `snowflake_workspace`

| | snowflake_workspace | doris_workspace |
|---|---|---|
| Tables | ✅ | ✅ |
| Views | ✅ | ✅ |
| Materialized Views | ✅ (DT, MV, Iceberg) | ✅ (async MV) |
| Streams (CDC) | ✅ | ❌ (Doris has Routine Load — see `doris_routine_load_sensor`) |
| Stages | ✅ | ❌ |
| Alerts | ✅ | ❌ |
| OpenFlow | ✅ | ❌ |
| Tasks | ✅ (`EXECUTE TASK USING CONFIG`) | ❌ (Doris's scheduled jobs are out-of-band) |
| Stored procedures | ✅ | ❌ (Doris doesn't yet support stored procedures) |

The narrower surface keeps this component ~10× smaller than `snowflake_workspace` (~350 LOC vs 2200+).
