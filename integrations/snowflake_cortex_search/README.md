# Snowflake Cortex Search

Query a Snowflake **Cortex Search Service** (RAG / vector search) and materialize the results as a Dagster asset.

## Purpose

Snowflake Cortex Search is a fully-managed RAG / vector-search engine inside Snowflake — you point it at a table of documents and it handles chunking, embedding, indexing, and serving. This component issues a query against an existing search service and writes the top-N results back as a Dagster asset.

Useful for:

- **Retrieval-quality evals** — same query against the same service, materialized on a schedule, so you can audit how results drift as the corpus changes
- **Snapshot-as-an-asset** — capture today's "top 10 results for X" as an immutable point-in-time record downstream consumers can read
- **Search-as-a-feature** — the search result list becomes input to a downstream Dagster asset (LLM completion, ranking pipeline, dashboard backend)

## Prerequisites

A Cortex Search Service must exist in the target schema. Create one via SQL:

```sql
CREATE OR REPLACE CORTEX SEARCH SERVICE my_docs_search
  ON content TARGET_LAG = '1 hour' WAREHOUSE = COMPUTE_WH
  AS SELECT id, title, content, category FROM DAGSTER_DEMO.AI.DOCS;
```

Then point this component at it via the `service_name` field.

## Example YAML
```yaml
type: dagster_component_templates.SnowflakeCortexSearchComponent
attributes:
  asset_name: cortex_search_results
  account:     "{{ env('SNOWFLAKE_ACCOUNT') }}"
  user:        "{{ env('SNOWFLAKE_USER') }}"
  database:    DAGSTER_DEMO
  schema_name: AI
  authenticator: SNOWFLAKE_JWT
  private_key_file: "{{ env('SNOWFLAKE_PRIVATE_KEY_FILE') }}"

  service_name: my_docs_search
  query: "how do I configure Snowpipe auto-ingest from S3?"
  result_limit: 10
  result_columns: [ID, TITLE, CONTENT]
```

## Returned asset value

The asset's value is a list of result dicts: `[{ID: ..., TITLE: ..., CONTENT: ...}, ...]`. Materialization metadata includes:

| Key | Description |
|---|---|
| `cortex/service` | Fully-qualified service name |
| `cortex/query` | The search query that was issued |
| `cortex/result_count` | Number of results returned |
| `cortex/top_5_preview` | Markdown preview of the first 5 results |

## Filter syntax

`result_filter` accepts a JSON filter string. See [Cortex Search filter syntax](https://docs.snowflake.com/en/user-guide/snowflake-cortex/cortex-search/query-cortex-search-service#filter-syntax). Common shapes:

- `'{"@eq": {"category": "docs"}}'` — exact-match
- `'{"@contains": {"tags": "billing"}}'` — array contains
- `'{"@and": [{"@eq": {"team": "platform"}}, {"@gte": {"updated_at": "2025-01-01"}}]}'` — compound
