# Snowflake Iceberg Table

Define a Snowflake **ICEBERG TABLE** as Dagster YAML. Materialization runs `CREATE OR REPLACE ICEBERG TABLE` against your Snowflake account; the resulting table is registered as a Dagster asset with full lineage, retries, partitions, freshness, and catalog metadata.

## Purpose

Iceberg tables are Snowflake's open-table-format integration — data is stored in cloud object storage (S3 / GCS / Azure) as Apache Iceberg files, but Snowflake reads + writes natively. You get the warehouse's compute on truly portable data: Spark / Trino / Athena / DuckDB / dbt / Polars can all read the same files.

Two flavors of Iceberg in Snowflake — both supported by this component via the `catalog` field:

- **Snowflake-managed** (`catalog: SNOWFLAKE`) — Snowflake owns the metadata catalog. Simplest path; no external catalog needed.
- **External catalog** (`catalog: GLUE` / `POLARIS` / `UNITY` / `TABULAR` / your registered catalog name) — metadata lives in Glue / Polaris / Unity / Tabular and is read by Snowflake.

Pairs with `snowflake_workspace` (which DISCOVERS existing entities) — use this component when you want to define new Iceberg tables as code rather than maintain them in Snowflake worksheets.

## Prerequisites

1. **EXTERNAL VOLUME** registered with the cloud-storage location:
   ```sql
   CREATE OR REPLACE EXTERNAL VOLUME MY_S3_VOLUME
     STORAGE_LOCATIONS = (
       (NAME = 'my-aws-s3', STORAGE_PROVIDER = 'S3',
        STORAGE_BASE_URL = 's3://my-bucket/iceberg/',
        STORAGE_AWS_ROLE_ARN = 'arn:aws:iam::ACCOUNT:role/snowflake-iceberg')
     );
   ```
2. **CREATE ICEBERG TABLE privilege** on the target schema.

## Example YAML
```yaml
type: dagster_component_templates.SnowflakeIcebergTableComponent
attributes:
  asset_name: orders_iceberg
  account:     "{{ env('SNOWFLAKE_ACCOUNT') }}"
  user:        "{{ env('SNOWFLAKE_USER') }}"
  database:    DAGSTER_DEMO
  schema_name: ANALYTICS
  authenticator: SNOWFLAKE_JWT
  private_key_file: "{{ env('SNOWFLAKE_PRIVATE_KEY_FILE') }}"

  table_name:      orders_iceberg
  external_volume: MY_S3_VOLUME
  catalog:         SNOWFLAKE
  base_location:   iceberg/orders/

  sql: |
    SELECT * FROM DAGSTER_DEMO.RAW.ORDERS WHERE STATUS = 'completed'

  cluster_by: [ORDER_DATE]
```
