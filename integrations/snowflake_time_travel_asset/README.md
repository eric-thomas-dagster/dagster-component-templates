# Snowflake Time Travel Asset

Query an existing Snowflake table **as of a past point in time** (Time Travel) and materialize the result as a Dagster asset.

## Purpose

Snowflake's Time Travel lets you SELECT against any table at a past state — within the table's data-retention window (default 1 day on Standard, configurable up to 90 days on Enterprise+). This component lifts that capability into a first-class Dagster asset, so you can:

- **Audit** what a table looked like before an incident (point-in-time snapshot, queryable downstream, immutable in the catalog)
- **Reproduce** pipeline output from a specific historical state ("rerun yesterday's marketing rollup against yesterday's RAW.ORDERS")
- **Compare** current vs. historical values via a side-by-side DataFrame in `dg dev`

## Three time-travel modes

Set exactly one:

| Field | Snowflake clause | Example |
|---|---|---|
| `time_travel_offset_seconds` | `AT(OFFSET => -<n>)` | `-3600` → 1 hour ago |
| `time_travel_timestamp` | `AT(TIMESTAMP => '...'::TIMESTAMP)` | `'2025-05-19 10:00:00'` |
| `time_travel_statement_id` | `BEFORE(STATEMENT => '<query_id>')` | Snowflake query-history ID |

## Example YAML
```yaml
type: dagster_component_templates.SnowflakeTimeTravelAssetComponent
attributes:
  asset_name: orders_one_hour_ago
  account:     "{{ env('SNOWFLAKE_ACCOUNT') }}"
  user:        "{{ env('SNOWFLAKE_USER') }}"
  database:    DAGSTER_DEMO
  schema_name: RAW
  authenticator: SNOWFLAKE_JWT
  private_key_file: "{{ env('SNOWFLAKE_PRIVATE_KEY_FILE') }}"

  source_table: ORDERS
  columns: [ORDER_ID, CUSTOMER_ID, AMOUNT, STATUS]
  time_travel_offset_seconds: -3600
  where_clause: "STATUS = 'completed'"
  row_limit: 1000
```

## Returned value

The asset returns a pandas DataFrame. Materialization metadata includes:

| Key | Description |
|---|---|
| `snowflake/sql` | The exact SQL that was issued |
| `snowflake/source_table` | Fully-qualified table |
| `snowflake/row_count` | Rows in the result |
| `snowflake/preview` | First 10 rows as a Markdown table |

## Retention caveat

Time Travel only works within the table's `DATA_RETENTION_TIME_IN_DAYS`. If you ask for an offset beyond retention, Snowflake returns a clear error. Check / extend retention via:

```sql
ALTER TABLE my_table SET DATA_RETENTION_TIME_IN_DAYS = 30;
```
