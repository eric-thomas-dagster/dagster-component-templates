# Firebase Resource

Register a Firebase Admin SDK resource for use by other components (Firestore, Cloud Storage, Realtime Database, Cloud Functions callers).

## Installation

`pip install firebase-admin`

```yaml
attributes:
  resource_key: firebase_resource
  project_id_env_var: FIREBASE_PROJECT_ID
  credentials_path_env_var: FIREBASE_CREDENTIALS_PATH
  # Or, alternatively (mutually exclusive):
  # credentials_json_env_var: FIREBASE_CREDENTIALS_JSON
```

## Authentication

Firebase Admin uses a Google service account. Set exactly one of:

- `credentials_path_env_var` — env var holds a filesystem path to a JSON key.
- `credentials_json_env_var` — env var holds the raw JSON contents (useful in
  container / CI environments where mounting files is inconvenient).

Setting both — or neither — raises a `ValueError` at load time.

Create and download a service account key here:
https://console.firebase.google.com/project/_/settings/serviceaccounts/adminsdk

```bash
export FIREBASE_PROJECT_ID="my-firebase-project"
export FIREBASE_CREDENTIALS_PATH="/secrets/firebase-admin.json"
# OR
export FIREBASE_CREDENTIALS_JSON="$(cat /secrets/firebase-admin.json)"
```

## Idempotency

Firebase Admin's `initialize_app` raises if you call it twice with the same
app name. The resource caches the `App` instance keyed by `resource_key`, so
repeat `get_app()` / `get_firestore()` / `get_storage()` calls across assets
in the same process are safe.

## Provides

- `get_app()` — the underlying `firebase_admin.App`
- `get_firestore()` — a Firestore `Client`
- `get_storage(bucket_name=None)` — a Cloud Storage `Bucket`
- `get_realtime_db(path="/")` — a Realtime Database `Reference`

## Usage in other components

```yaml
attributes:
  resource_name: firebase_resource
```

Firebase-aware components in this registry (`firestore_collection_ingestion`,
`firebase_storage_monitor`) look up the resource by name.
