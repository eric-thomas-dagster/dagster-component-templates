# Gong Resource

Register a `GongResource` wrapping Gong API v2 Basic Auth credentials for use by other Dagster components (ingestion, transcript-analysis, etc.).

## Authentication

Gong uses HTTP Basic Auth with a two-part credential (`access_key` + `access_key_secret`).
Both fields accept an **environment variable name** (not the secret values themselves).
Set the corresponding env vars before running Dagster:

```bash
export GONG_ACCESS_KEY="your-access-key"
export GONG_ACCESS_KEY_SECRET="your-access-key-secret"
```

Generate credentials from the Gong web app: Company Settings > Ecosystem > API.

## Client API

`resource.get_client()` returns a `GongClient` with:

- `client.base_url` — resolved base URL (default `https://api.gong.io/`)
- `client.session` — a `requests.Session` with Basic Auth pre-applied
- `client.get(path, **kwargs)` / `client.post(path, **kwargs)` — call any endpoint by relative path (e.g. `client.get("v2/calls", params={...})`)
