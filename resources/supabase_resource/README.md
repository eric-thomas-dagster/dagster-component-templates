# Supabase Resource

Register a `SupabaseResource` wrapping the [supabase-py](https://github.com/supabase-community/supabase-py) client for use by other Dagster components (storage monitor, edge function trigger, vector search, etc.).

## Installation

```bash
pip install "supabase>=2.0.0" "requests>=2.28"
```

## Authentication

Both env vars hold **names** (not values). Set them before running Dagster:

```bash
export SUPABASE_URL="https://xxxxxxxxxxxxxxxx.supabase.co"
export SUPABASE_KEY="eyJhbGciOi..."
```

Get keys from the Supabase Dashboard → *Project Settings* → *API*:
- **anon (public)** — safe for browser use, respects RLS
- **service_role (secret)** — bypasses RLS; treat like a database password

Use the service-role key for asset ingestion, storage admin, and vector-search components. Never expose the service-role key to a browser.

## Notes

`SupabaseResource.get_client()` returns a `supabase.Client` initialized via `supabase.create_client(url, key)`. The client exposes `.table(...)`, `.storage`, `.functions`, `.rpc(...)`, and `.auth` sub-clients per the official supabase-py surface.
