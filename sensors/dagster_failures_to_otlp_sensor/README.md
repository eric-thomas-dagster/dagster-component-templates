# dagster_failures_to_otlp_sensor

Push **Dagster run failure events** as OTLP/HTTP log records to any OpenTelemetry-compatible collector or backend ‚Äî Honeycomb, Datadog OTel-ingest, Grafana Loki via OTel Collector, ClickHouse via OTel, AWS CloudWatch OTel, etc.

Operates as a `@run_failure_sensor`. On each failure, builds a hand-rolled OTLP/HTTP logs envelope (JSON body) and POSTs to `{endpoint}/v1/logs`. No OpenTelemetry SDK dependency ‚Äî just `requests`.

This is intentionally narrower than the other telemetry sensors:
- **Failures only** (not all run lifecycle events) ‚Äî the use case is alerting / triage, not metrics
- **Log shape** (not metric shape) ‚Äî fits the OTel logs pipeline naturally

For metric-shaped run telemetry use [`dagster_runs_to_prometheus_sensor`](../dagster_runs_to_prometheus_sensor/) or [`dagster_runs_to_influxdb_sensor`](../dagster_runs_to_influxdb_sensor/).

## OTLP record shape

Per OTLP/HTTP logs spec ‚Äî one ResourceLog with one ScopeLog with one LogRecord per failure:

```json
{
  "resourceLogs": [{
    "resource": {
      "attributes": [
        {"key": "service.name", "value": {"stringValue": "dagster"}},
        {"key": "service.instance.id", "value": {"stringValue": "prod-east"}}
      ]
    },
    "scopeLogs": [{
      "scope": {"name": "dagster_community_components.dagster_failures_to_otlp_sensor"},
      "logRecords": [{
        "timeUnixNano": "1748534567000000000",
        "severityNumber": 17,
        "severityText": "ERROR",
        "body": {"stringValue": "Dagster run abc-123 for job ingest_orders failed. step=load_to_warehouse err=..."},
        "attributes": [
          {"key": "dagster.job_name", "value": {"stringValue": "ingest_orders"}},
          {"key": "dagster.run_id", "value": {"stringValue": "abc-123-uuid"}},
          {"key": "dagster.duration_seconds", "value": {"doubleValue": 42.7}},
          {"key": "dagster.failure_step", "value": {"stringValue": "load_to_warehouse"}}
        ]
      }]
    }]
  }]
}
```

## Example ‚Äî Honeycomb

```yaml
type: dagster_component_templates.DagsterFailuresToOtlpSensorComponent
attributes:
  sensor_name: dagster_failures_to_honeycomb
  otlp_endpoint_env_var: HONEYCOMB_OTLP_ENDPOINT   # https://api.honeycomb.io
  otlp_headers_env_var: HONEYCOMB_OTLP_HEADERS     # x-honeycomb-team=<key>
  service_name: dagster
  location_label: prod-east
```

Required env:

```
HONEYCOMB_OTLP_ENDPOINT=https://api.honeycomb.io
HONEYCOMB_OTLP_HEADERS=x-honeycomb-team=<your_api_key>,x-honeycomb-dataset=dagster_failures
```


## Example ‚Äî OTel Collector ‚Üí Grafana Loki

```yaml
type: dagster_component_templates.DagsterFailuresToOtlpSensorComponent
attributes:
  sensor_name: dagster_failures_to_loki
  otlp_endpoint_env_var: OTEL_COLLECTOR_ENDPOINT   # http://otel-collector.svc:4318

```

The OTel Collector handles the Loki adapter side; this component just speaks vanilla OTLP/HTTP.
