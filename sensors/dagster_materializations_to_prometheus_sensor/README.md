# dagster_materializations_to_prometheus_sensor

Push **per-asset materialization metrics** to Prometheus Push Gateway / VictoriaMetrics / Mimir-via-gateway. Companion to [`dagster_runs_to_prometheus_sensor`](../dagster_runs_to_prometheus_sensor/), which operates at the job level — this one tracks individual asset materializations.

Two metrics per materialization:

| Metric | Type | Labels |
|---|---|---|
| `dagster_asset_materialization_count` | counter | asset_key, location |
| `dagster_asset_last_materialization_timestamp` | gauge (unix seconds) | asset_key, location |

Useful for Grafana dashboards like "which critical assets were last materialized > 24h ago" — query the `last_materialization_timestamp` gauge, compare to `time()`, alert on staleness.

## Two implementation modes

| Mode | Trigger | When |
|---|---|---|
| Filtered (explicit keys) | `@multi_asset_sensor` — Dagster's native API, only fires on materializations of monitored assets | `watch_asset_keys: [...]` is set |
| Workspace-wide | Generic `@sensor` that polls the event log via `instance.get_event_records()` with a cursor | `watch_asset_keys` is None |

Filtered mode is more efficient at high asset counts since Dagster's framework handles the filtering. Workspace-wide is simpler for ad-hoc setups.

## Example — workspace-wide push to Push Gateway

```yaml
type: dagster_component_templates.DagsterMaterializationsToPrometheusSensorComponent
attributes:
  sensor_name: dagster_asset_materializations
  base_url_env_var: PROM_PUSHGATEWAY_URL
  target_kind: push_gateway
  push_job: dagster_assets
  location_label: prod-east
```

## Example — filtered to critical assets, push to VictoriaMetrics

```yaml
type: dagster_component_templates.DagsterMaterializationsToPrometheusSensorComponent
attributes:
  sensor_name: critical_asset_freshness
  base_url_env_var: VM_BASE_URL
  target_kind: vm_import
  watch_asset_keys:
    - orders_clean
    - revenue_daily_summary
  location_label: prod-east
```

## Sample Grafana query

Find assets stale > 24h:

```promql
time() - dagster_asset_last_materialization_timestamp > 86400
```
