# dagster_runs_to_influxdb_sensor

Push **Dagster's own run lifecycle metrics** (duration, status, run_id) to [InfluxDB](https://www.influxdata.com/) 2.x / 3.x via the `/api/v2/write` line-protocol endpoint on every run completion. Also works against **AWS Timestream for InfluxDB** — same SDK, same wire protocol.

Operates as two `@run_status_sensor` instances (SUCCESS + FAILURE). On each event, POSTs one line-protocol point to InfluxDB. Distinct from the data-plane `dataframe_to_influxdb` sink — this is *operational telemetry*, not user data.

## Line protocol shape

```
dagster_run,job_name=ingest_orders,status=success,location=prod-east \
  duration_seconds=42.7,run_id="abc-123-uuid",finished_ts=1748534567i \
  1748534567000000000
```

- **Tags** (indexed, low-cardinality): `job_name`, `status`, `location`
- **Fields** (data, not indexed): `duration_seconds` (float), `run_id` (string), `finished_ts` (int — unix seconds)
- **Timestamp**: unix nanoseconds (line-protocol native precision)

## Example

```yaml
type: dagster_component_templates.DagsterRunsToInfluxdbSensorComponent
attributes:
  sensor_name: dagster_runs_to_influxdb
  base_url_env_var: INFLUX_URL
  token_env_var: INFLUX_TOKEN
  org: my-org
  bucket: dagster-runs
  measurement: dagster_run
  location_label: prod-east
```

Required env: `INFLUX_URL`, `INFLUX_TOKEN`.

## Querying back

```flux
from(bucket: "dagster-runs")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "dagster_run")
  |> filter(fn: (r) => r._field == "duration_seconds")
  |> group(columns: ["job_name", "status"])
  |> aggregateWindow(every: 5m, fn: mean)
```

```sql
-- InfluxDB 3.x SQL via Arrow Flight
SELECT job_name, status, AVG(duration_seconds) AS avg_dur
FROM dagster_run
WHERE time > now() - INTERVAL '1 hour'
GROUP BY job_name, status
```

## AWS Timestream for InfluxDB

Point `INFLUX_URL` at your Timestream-for-InfluxDB endpoint (`https://<id>-<region>.timestream-influxdb.<region>.on.aws:8086`). Same token + org + bucket model.
