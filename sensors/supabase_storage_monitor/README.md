# Supabase Storage Monitor

Watch a Supabase Storage bucket for new files and emit a `RunRequest` per new object. Cursor-tracked by the newest observed `updated_at`, so re-scans skip previously-seen files.

## Installation

```bash
pip install "supabase>=2.0.0"
```

Requires a `SupabaseResource` (see `resources/supabase_resource`) providing the client.

## Configuration

```yaml
type: dagster_component_templates.SupabaseStorageMonitorComponent
attributes:
  sensor_name: supabase_incoming_files_sensor
  bucket_name: incoming
  prefix: raw/
  resource_name: supabase_resource
  minimum_interval_seconds: 60
  default_status: stopped
```

## Behavior

- Walks the bucket recursively so nested folders are picked up.
- Skips folder placeholders (Supabase returns them with `id: None` and `metadata: None`).
- Cursor is the ISO timestamp of the newest observed `updated_at`.
- `run_key` = `<bucket>/<path>-<etag>` so a re-uploaded file with a new ETag triggers a fresh run.

When `dynamic_partition_name` is set, each new path is registered as a partition key of that dynamic partitions definition and the RunRequest carries the path as `partition_key`. Otherwise, object metadata is passed via `run_config.ops.config`.
